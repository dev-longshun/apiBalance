# Telegram Bot 命令契约：Check Balance

**运行模式**: Long Polling（启动时如果 bot_token 已配置则自动开始监听）

**通用规则**:
- 所有命令响应使用 Markdown 格式（ParseMode: MarkdownV2）
- 未识别的命令回复帮助提示
- 仅响应配置的 chat_id 发来的消息（忽略其他聊天）

---

## /balance

查询所有站点的最新余额。**实时查询**，不使用缓存数据。

**触发**: 用户发送 `/balance`

**Bot 行为**:
1. 立即回复 "正在查询所有站点余额..."（即时反馈）
2. 并发查询所有站点的实时余额（受并发上限控制）
3. 查询完成后，编辑上述消息为最终结果（或发送新消息）

**响应示例**:
```
📊 额度总览

🟢 Example API      $45.23
🟢 Forward API      $128.50
🟡 Sub2API          $8.20  ⚠️ 低于阈值
🔴 Test Site        查询失败

总计: 2 正常 / 1 告警 / 1 异常
查询时间: 2026-06-08 15:35:00
```

**状态图标规则**:
- 🟢 status=ok
- 🟡 status=low
- 🔴 status=error
- ⚪ status=unknown

---

## /balance <站点名>

查询指定站点的详细最新余额。**实时查询**。

**触发**: 用户发送 `/balance Example API`

**匹配规则**: 站点名称模糊匹配（包含即匹配，不区分大小写）。若匹配到多个站点，列出匹配项让用户选择。

**Bot 行为**:
1. 立即回复 "正在查询 Example API..."
2. 实时查询该站点余额
3. 编辑消息为详细结果

**响应示例**:
```
📋 站点详情: Example API

💰 余额: $45.23 (USD)
📡 API 格式: OpenAI 兼容
🔔 告警阈值: $50, $10
   ├ $50: ⚠️ 已触发
   └ $10: ✅ 未触发
🕐 查询时间: 2026-06-08 15:35:00
```

**错误**: 未找到匹配站点时回复 "未找到名称包含 'xxx' 的站点。"

---

## /refresh

立即触发一轮全量查询并更新仪表盘数据。

**触发**: 用户发送 `/refresh`

**Bot 行为**:
1. 回复 "正在刷新所有站点余额..."
2. 触发全量查询（与定时轮询逻辑相同）
3. 完成后回复结果摘要（格式同 `/balance`）

**防重入**: 如果当前已有轮询任务在运行，回复 "当前已有查询任务在进行中，请稍后再试。"

## /help

显示帮助信息。

**触发**: 用户发送 `/help` 或任何未识别的命令

**响应**:
```

/balance        - 实时查询所有站点余额
/balance <名称>  - 实时查询指定站点余额
/refresh        - 刷新所有站点并更新面板
/status         - 查看系统运行状态
/help           - 显示本帮助信息
```

## 告警通知（被动推送）

非用户命令触发，由轮询周期中阈值逻辑驱动。

**触发条件**: 轮询后某站点 `balance < threshold.amount` 且 `threshold.triggered == 0`

**消息格式**:
```
⚠️ 余额不足提醒

站点: Example API
当前余额: $4.52
触发阈值: $10.00
查询时间: 2026-06-08 15:30:00

请及时充值，避免服务中断。
```
